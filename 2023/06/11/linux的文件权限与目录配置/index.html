linux的文件权限与目录配置

用户组

  • 添加用户组：groupadd -g 550（id） test（组名）
  • 删除用户组：groupdel test (组名)
  • 用户组位置：/etc/group
  • 修改 test 文件夹用户组为 guanbin: chgrp -R guanbin test/

用户

  • 添加用户：useradd test
  • 设置用户密码：passwd 123
  • 用户位置：/etc/passwd
  • 密码位置：/etc/shadow
  • 修改 test 文件夹用户为 test: chown -R test test/

其他

  • 修改所有者和用户 chown root:root test/

修改权限

  • chmod -[R] 文件或目录
  • 可读4+可写2+可执行1=7
  • chmod u/g/o/a 分表代表，user/group/other/all
  • chmod u=rwx,go=rw test.sh //表示用户为 rwx 用户组合其他权限为 rw
  • chmod a+w test.sh //表示文件对所用用户加上可写权限
  • chmod a-w test.sh //表示对所有用户去掉科协权限
  • chmod 777 // 表示对用户，用户组，及其他用户都有 rwx 权限

文件夹的权限释义

  • r 表示可以看到此目录
  • w 表示可以删除，修改此目录和文件
  • x 表示可以 cd 到此目录内

查询命令

  • ls -al //查询出所有文件，包括隐藏文件

切换文件夹

  • cd ../文件夹 切到上一个文件夹
  • cd - 回到刚才的目录，相当于返回上一步的操作
  • cd ~ 回到自己的主文件夹

CP 命令

  • cp -i 源文件 目标文件 //会询问是否覆盖（已经有同名的情况下）
  • cp -r 递归复制
  • cp 文件夹or 文件 . //拷贝到当前目录下

basename

  • basename /root/test/1.txt -> 输出为1.txt
  • dirname /root/test/1.txt -> 输出为 /root

查看文件命令

  • cat -n 文件 // 显示行号
  • tac -n //是按倒序显示
  • nl -b //如果有空行，空行的不列出行号
  • more 文件， 空格 显示下一页； enter 键显示下一行， b 显示上一页， q 退出， :f 显示当前行及文件名
  • less

  • head -n 文件 //取前几行

  • tail -n 文件 //取文件后几行

  • head -n 20 /etc/man_db.conf | tail -n 10 // 取文件的第11到20行

  • od 查看非文本文件 -c 即以 ascii 方式查看
    od -t c /usr/bin/passwd

touch

  • ls -l 默认显示的是修改时间

  • ls -l –time=atime 文件 显示文件被使用时的时间，使用 cat时就会更新这个时间

  • ls -l –time=ctime 文件 权限和属性被更改了，会更新这个时间

  • touch -d “2 days ago” 文件名 更新 mtime,atime 为两天前的时间

  • ll haha.txt ; ll –time=atime haha.txt ; ll –time=ctime haha.txt //;代表连续命令的执行

umask

umask 022 或 unmask -S 即为用户没去掉权限，用户组合 other 去掉了，可写权限

新建文件默认权限为 666 ，若 umask 为022 即去掉了科协权限，就变成了644
新建文件夹默认权限为777，若 umask 为022 即去掉了科协权限，就变成了755

chattr

  • chattr +i testhaha //设置文件不能被删除，修改，删除等， 解除限制，chattr -i testhaha

lsattr

  • 作用是把chattr设置的属性，给显示出来

查找命令

  • whereis 和 locate 是查找数据库中的文件，速度比 find 命令快，因为没有直接查找硬盘，数据库中的文件，基本上是每天执行一次刷新，故使用 whereis 和 locate 时会找到已经删除的文件，或者刚刚新建的文件找不到。
  • updatedb 手动执行一次刷新数据库的操作
  • find /home -user test // 找出home 目录下，属于 test 的所有文件
  • find / -nouser // 找出所有不属于任何人的文件
  • find / -name hahalocate 查找所有文件中，是 hahalocate 的文件

  • tcpdump -i any tcp and host 118.19.136.37 and port 80 -w diu.pacp

  • 挥手可能是三次，如果服务端没有其他事情处理，第二次 fin+ack 服务端会合并在一起发送给客户端

  • 丢包，超时重传的相隔时间是每次指数形式递增的，查看系统最多重试次数：cat /proc/sys/net/ipv4/tcp_syn_retries

  • iptables -I INPUT -p tcp –dport 80 -j REJECT //禁止访问80端口

  • iptables -I INPUT -p tcp –dport 80 -j ACCEPT //打开访问80端口

  • 一次 http 请求，会有2.5个 RTT 的时延，或者2个（http 请求和第三次握手的 ack 一起发给客户端），若 在 Linux 3.7 内核版本中，提供了 TCP Fast Open 功能，这个功能可以减少 TCP 连接建立的时延，可以通过设置 net.ipv4.tcp_fastopen 内核参数，来打开 Fast Open 功能。 第二次请求时，带上第一次的 cookie,客户验证 cookie 的时效性，可以减少握手的次数
